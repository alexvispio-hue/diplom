# Архитектура приложения

## Назначение

Приложение предназначено для распознавания рукописного текста на изображениях. Система строится как модульный OCR pipeline, где каждый этап можно развивать независимо: загрузка изображения, предобработка, инференс локальной нейросетевой модели, постобработка текста и сохранение результата.

## Компоненты

### UI

Пользовательский интерфейс реализуется на Streamlit. Он нужен для демонстрации дипломного проекта: загрузка изображения, предпросмотр, запуск распознавания, экспорт результата, отображение истории и описание экспериментальной оценки.

### API

FastAPI предоставляет программный интерфейс:

- `GET /api/health` — проверка состояния;
- `GET /api/models` — информация о подключенной OCR-модели;
- `POST /api/recognize` — загрузка изображения и запуск распознавания;
- `POST /api/recognize-page` — распознавание страницы с автоматическим выделением строк;
- `GET /api/history` — список последних результатов;
- `GET /api/history/{id}` — отдельный результат распознавания.
- `GET /api/history/{id}/image/original` — исходное изображение;
- `GET /api/history/{id}/image/processed` — изображение после предобработки.

### RecognitionService

Сервис координирует полный сценарий:

1. сохраняет исходный файл;
2. проверяет размер загруженного файла;
3. при выбранном режиме запускает предобработку;
4. вызывает OCR-модель;
5. замеряет время обработки;
6. сохраняет результат в базу данных.

Для режима страницы сервис дополнительно запускает `PageLineSegmenter`, распознает найденные строки по очереди и объединяет результат с переносами строк.

### ImagePreprocessor

Модуль предобработки использует OpenCV:

- чтение изображения;
- перевод в оттенки серого;
- удаление шумов bilateral-фильтром;
- нормализация контраста через CLAHE;
- сохранение подготовленного изображения.

### PageLineSegmenter

Модуль сегментации страницы использует OpenCV:

- повышает локальный контраст;
- бинаризует изображение;
- удаляет длинные линии тетради;
- находит горизонтальные полосы текста;
- сохраняет отдельные изображения строк и размеченную копию страницы.

### OCR

Основной распознаватель — `TrOCRRecognizer`. Он использует:

- `TrOCRProcessor`;
- `VisionEncoderDecoderModel`;
- PyTorch;
- локальный checkpoint `models/trocr-cyrillic-finetuned`, если он доступен;
- резервную русскоязычную модель `kazars24/trocr-base-handwritten-ru`.

Распознавание выполняется локально. Внешний OCR API не используется.

### Metrics

Модуль `app.ocr.metrics` содержит функции для экспериментальной оценки:

- расстояние Левенштейна;
- Character Error Rate;
- Word Error Rate.

Эти функции нужны для сравнения эталонного текста и результата распознавания на тестовом наборе.

### Evaluation

`EvaluationService` выполняет пакетную оценку по CSV-манифесту. Для каждой строки он:

1. читает изображение;
2. выполняет предобработку;
3. запускает распознавание;
4. сравнивает результат с эталонным текстом;
5. записывает CER/WER и время обработки в итоговый CSV.

### Storage

История хранится в SQLite. Для каждой обработки сохраняются:

- исходное имя файла;
- размер исходного файла;
- путь к исходному изображению;
- путь к обработанному изображению;
- распознанный текст;
- название модели;
- время обработки;
- дата создания записи.
- применялась ли предобработка.
- режим распознавания: фрагмент или страница;
- количество найденных строк;
- пути к изображениям найденных строк.

## Расширяемость

OCR-модуль построен через базовый интерфейс `BaseRecognizer`. Это позволяет добавить другие локальные модели:

- CRNN + CTC Loss;
- fine-tuned TrOCR;
- специализированную модель для русского рукописного текста.

Такой подход подходит для дипломной работы, потому что поддерживает как прикладное приложение, так и экспериментальную часть с сравнением моделей.
