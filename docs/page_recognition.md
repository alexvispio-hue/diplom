# Распознавание страниц

## Реализованный pipeline

В приложении есть два режима:

1. `Фрагмент строки` передает одно изображение локальной TrOCR-модели.
2. `Страница` выделяет строки на фотографии, распознает их по очереди и объединяет результат с переносами строк.

Первая версия сегментации реализована в `app/image_processing/page_segmentation.py` с помощью OpenCV. Она предназначена для ровных фотографий или сканов листа с горизонтальными строками. В истории сохраняется размеченная копия страницы, поэтому качество выделения строк можно проверить визуально.

## Следующий нейросетевой этап

Для сложных страниц следует подключить `Kraken` и обучить его baseline-сегментатор. Эта библиотека подходит для задачи, потому что ее сегментатор ищет строки и области текста на странице, поддерживает рукописные документы и может дообучаться на собственных данных.

Для обучения потребуются не вырезанные строки из текущего датасета, а фотографии страниц с разметкой линий текста в формате PAGE XML или ALTO XML. Практический порядок работы:

1. собрать репрезентативные фотографии конспектов;
2. разметить линии текста на страницах;
3. разделить страницы на обучающую и проверочную выборки;
4. дообучить baseline-сегментатор Kraken на GPU;
5. заменить эвристический `PageLineSegmenter` адаптером обученной модели;
6. визуально сравнить выделение строк на фотографиях, которых не было в обучении.

Минимальная команда Kraken для обучения на GPU:

```bash
ketos -d cuda segtrain -f xml training_data/*.xml
```

## Границы текущей версии

Текущий OpenCV-сегментатор не следует считать итоговой нейросетевой моделью. Он нужен как рабочая первая версия приложения и как ориентир для сбора страниц, на которых выделение строк требуется улучшить.

Официальная документация:

- [Kraken: Page Segmentation](https://kraken.re/main/advanced/segmentation.html)
- [Kraken: Segmentation Training](https://kraken.re/6.0.0/training/segtrain.html)
