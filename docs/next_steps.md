# План дальнейшей разработки

## Ближайшие задачи

1. Подобрать или подготовить модель для русского рукописного текста.
2. Собрать небольшой тестовый набор изображений с эталонными расшифровками.
3. Провести пакетную оценку CER/WER на реальных изображениях.
4. Добавить экспорт результата в DOCX.
5. Подготовить таблицы и графики для главы с экспериментами.

## Экспериментальная часть

Для диплома важно показать не только работу приложения, но и измеримую оценку качества. Минимальный эксперимент:

- 20-50 изображений рукописного текста;
- эталонная расшифровка для каждого изображения;
- запуск OCR pipeline через `scripts/evaluate_dataset.py`;
- расчет CER/WER;
- таблица результатов;
- выводы по типичным ошибкам модели.

## Модель

Текущая `microsoft/trocr-base-handwritten` подходит для проверки архитектуры, но плохо подходит для русского рукописного текста. Для дипломного качества нужно:

- найти TrOCR/Transformer OCR модель с поддержкой кириллицы;
- либо дообучить модель на русскоязычном наборе;
- либо добавить CRNN+CTC как собственную архитектуру для сравнения.
